## 第 1 课 — 多组学 / PAM50 / 批次效应（理解亚型体检的最小知识）

### 1.1 为什么"乳腺癌"要分亚型
"乳腺癌"不是一种病，而是分子层面差异巨大的一组病：有的激素治疗有效、预后好；有的（三阴性）凶险无靶向药。
**分子亚型 = 把"一种病"拆成治疗/预后不同的子群**。"发现亚型"的临床意义即在此，也是论文价值锚。

### 1.2 多组学（multi-omics）—— 用 CS 类比
中心法则：**DNA →(转录)→ RNA →(翻译)→ 蛋白质**。每层可测量 = 一个"组学层"。同一批病人多层测量 = **多模态学习**。

| 组学层 | 测什么 | CS 类比 | 矩阵形态 |
|---|---|---|---|
| 基因表达 (RNA-seq) | 每个基因此刻多活跃 | 哪些代码模块在跑、跑多频 | 样本×基因(~2万)，值=表达量 |
| DNA 甲基化 | 表观"开关"，把基因调静音 | feature flag / 注释掉代码 | 样本×CpG 位点 |
| 拷贝数变异 (CNV) | 某基因区被扩增/缺失几份 | 某模块被复制/删除 | 样本×基因区段 |

直觉：**DNA 在每个细胞基本相同（源码一样），但"哪些基因在跑、跑多猛"（表达）才区分亚型** → 表达层信息量最大，先用它。

### 1.3 PAM50 —— 我们的"金标准标签"（验证锚）
用 50 个基因把乳腺癌分成 **5 个公认亚型**的成熟方法。它是临床/文献承认的标准，我们拿它当**弱标签**验证聚类是否抓到真生物学（算 ARI/NMI）。

| PAM50 亚型 | 粗略含义 |
|---|---|
| Luminal A / B | 激素受体阳性；A 预后较好、B 较差 |
| HER2-enriched | HER2 基因驱动，有靶向药 |
| Basal-like | 多为三阴性，侵袭性强 |
| Normal-like | 接近正常组织 |

注意：PAM50 本身也是一种聚类结论、不是绝对真理。我们**不复刻它**，而是用它做合理性检验；创新在于更稳、更可解释、甚至它没覆盖的新亚型。

### 1.4 批次效应（batch effect）= 生物学版"域偏移"
TCGA 样本来自几十家医院（**TSS, tissue source site**）、不同测序板、不同试剂批次，各留**技术指纹**。
CS 类比：**dataset/domain shift**（像模型学会识别水印而非内容）。
**危险**：不处理则聚类会**按医院/批次分群而非生物学**。体检必查："簇是否与批次强相关？"

## 第 2 课 — 读懂真实 TCGA-BRCA 体检结果（紧扣 W1 数字）

体检结论：956 样本，Hopkins 0.694，ARI(KMeans,PAM50)=0.434 ≫ ARI(KMeans,TSS)=0.024 → **绿灯**。

### 2.1 为什么 262 个样本没 PAM50 标签、要剔除
TCGA 里除肿瘤外还有**癌旁正常组织**（条码 `-11`）和少数未分型样本，它们无 PAM50 调用。
真实组学数据天然"脏"——**先过滤到有金标准的队列再比较**是基本功。

### 2.2 PAM50 五型为何不均衡（LumA 434 ≫ Her2 67）
真实临床分布：Luminal A（激素受体阳性、预后最好）最常见；Her2-enriched、Basal（多三阴性、侵袭强）较少见。
**含义**：罕见亚型样本少、最难被聚类抓到 → 这是 **IP-4（开集/罕见新亚型发现）** 的机会；也说明**不能只看整体 ARI**（会被大类 LumA 主导）。

### 2.3 "ARI=0.434" 怎么用
这是**朴素 KMeans 探针**的成绩 = **基线地板（baseline to beat）**。目标**不是刷 ARI 到 1.0 复刻 PAM50**（无新意），
而是产出**更稳、更可解释、甚至 PAM50 没覆盖的亚型**。ARI 只是"没跑偏"的下限。

### 2.4 科学家的克制
簇-批次 ARI 低 ≠ 绝对无批次效应，簇**内部**仍可能有细微偏移。所以后续仍把 `tissue_source_site` 当协变量监控，而非从此无视。

## 第 3 课 — 多原型 / medoid / 质心陷阱（紧扣 W3 结果，含一个负结果与一个数据坑）

### 3.1 质心 vs medoid（核心概念）
- **质心 (centroid)**：一簇所有点的**平均**。问题：对长条/S形/流形簇，平均点可能落在**样本不存在的空白处**，
  不代表任何真实病人（lessons §5 "质心陷阱"）。高维组学更易中招。
- **medoid**：簇内**最居中的真实样本**（到簇内其它点总距离最小）。永不落空白 → 是一位**真实病人范例**，天然可解释。
- **多原型 (multi-prototype)**：一个亚型用**多个** medoid 代表（而非 1 个中心），以刻画复杂形状。
  CS 类比：与其用"一个平均向量"概括一类，不如用"几个真实代表样本"——像用几个典型用例描述一个聚类。

### 3.2 W3 的诚实负结果（重要）
我们在 BRCA 表达上比了 4 法，**多原型(IP-3) ARI 0.404 < KMeans 0.434 ≈ K-medoid 0.439**，没赢。
**这不是失败，是真信息**：说明 **BRCA 表达的亚型在降维空间相对"凸/团块"**，质心陷阱在这儿不严重 →
多原型的优势没机会发挥，反而其额外步骤带来噪声（方差更大）。
**教训（呼应你 lessons §7 的"对抗去域不是免费午餐"）**：**别假设一个高级方法一定更好，必须实测**。
→ 战略微调：对 BRCA 表达，**IP-1（稳定性）比 IP-3 更对症**；IP-3 可能要到**多组学融合**或非凸模态才显威力。

### 3.3 一个真实数据坑：`-11` 癌旁正常组织
聚类暴露：某个亚型的范例病人条码以 **`-11`** 结尾。TCGA 条码第 4 段是**样本类型码**：
**`-01` = 原发肿瘤**，**`-11` = 癌旁正常组织**（normal adjacent）。
PAM50 的 "Normal" 类**混入了真正的正常组织样本**。做**纯肿瘤亚型**时通常要**剔除 `-11`**——
否则你的"一个亚型"其实是"正常组织 vs 肿瘤"的平凡区分，而非肿瘤内部分型。下一步 W4 会剔除它重跑。

### 3.4 为什么要"多 seed + 子采样稳定性"
KMeans 等有随机初始化 → **单跑数字不可信**（lessons §8）。所以报 5 个种子的 **mean±std**。
更进一步，**子采样稳定性**（抽 80% 重聚类，看与全量结果是否一致，我们测得 0.474）是 **IP-1** 的雏形：
一个"真亚型"应当在**换一批病人/重采样下依然稳定**，不稳的多半是过拟合的伪亚型。

## 第 4 课 — 共识聚类的可复现性危机 / 零模型 / 乳腺癌的主分子轴（紧扣 W4）

### 4.1 为什么剔除正常组织后 ARI 反而下降（0.434 → 0.337）？
我们剔除了 `-11` 癌旁正常组织（PAM50 "Normal" 类 119→24，证实大多是正常组织）。结果 ARI 掉了——**这是好事、是诚实**：
- "肿瘤 vs 正常组织"差异巨大、**极易分**，它的存在把整体 ARI **虚高**了。
- 剔除后只剩"肿瘤内部亚型彼此区分"，本就**更微妙更难**。
**教训**：警惕被"平凡的易分项"抬高的指标——真正难、也真正有价值的是亚型间的细分。

### 4.2 共识聚类的"可复现性危机"（本项目要攻的痛点）
癌症亚型主流工具 **consensus clustering / ConsensusClusterPlus**（重采样多次、看哪些样本总聚在一起）被一篇著名论文
（**Șenbabaoğlu et al. 2014**）证明**过度乐观**：**即使数据没有真实结构，它也会"发现"看似稳定的簇并背书**。
根因：它**不和"零模型"比较**——只问"重采样下稳不稳"，不问"随机数据是不是也这么稳"。

### 4.3 零模型（null model）= 给稳定性一把"随机标尺"
**零模型**：把每个基因的数值在样本间**独立打乱**——保留每个基因自己的分布，但**摧毁基因之间的联合结构**（也就是簇结构）。
CS 类比：把表格每一列各自洗牌，行与行之间的"模式"就没了。
我们的 W4 结果：真实数据稳定性 **0.99**，零模型只有 **0.02**（贴地）。**真 ≫ null → 结构是真的**，不是 consensus clustering 那种自欺。这就是 **IP-1 的核心机制**。

### 4.4 k=2 是什么？乳腺癌的"主分子轴"
我们的方法（暂用 naive 选 k）认证出 **k\*=2**，且一簇由 **Basal** 主导。这在生物学上完全正确：
乳腺癌最主要的一刀是 **ER 阴性 / Basal-like（≈三阴性，侵袭强）** vs **ER 阳性 / Luminal**。
所以方法找到了**最稳健的那条真实生物学分界**。

### 4.5 但"按最大稳定性选 k"有偏置 → 下一步怎么改
稳定性曲线随 k **单调下降**：**越粗的划分天生越可复现**（k=2 永远最稳）。所以**不能简单按最大稳定性选 k**，
否则永远只给你 2 类、给不出临床在用的 5 类。
**W5 的更优思路**：不逼出单一 k，而是输出**"认证的亚型层级"**——k=2 高置信、k=5 仍显著高于 null、k=7 渐弱，
**逐层报告置信度**。这直接回应"亚型数到底取几"这个长期任意、没标准答案的痛点，比硬选一个 k 更诚实、也更有论文创新点。

## 第 5 课 — 置换检验 / p 值 / 多重检验 / "有没有结构"≠"分几型"（紧扣 W5）

### 5.1 置换检验 (permutation test) = 用"洗牌"造一把显著性尺子
W4 只比了一次 null；W5 把 null **做 R=20 次**（每次重新洗牌→重新算稳定性），得到稳定性的**零分布**。
- **经验 p 值** = 「随机数据稳定性 ≥ 真实稳定性」的比例。CS 类比：蒙特卡洛——洗很多次牌，看真实结果有多"出格"。
- **z 分数** = (真实 − null均值) / null标准差。我们测到 **z=89~239**（真实比随机高出近百个标准差）→ 结构极其确凿。
- 注意：p 最小只能到 1/(R+1)。R=20 → p 下限 0.048，所以我们所有 k 的 p 都"卡"在 0.048；**真正的强弱看 z**。正式论文会用 R≥1000。

### 5.2 多重检验校正 (BH/FDR) = 同时问很多问题要"打折"
我们对 5 个 k 各做一次检验。**做的检验越多，纯靠运气出现"显著"的概率越大**（CS 类比：多重比较/p-hacking）。
**Benjamini-Hochberg (BH)** 把 p 值按秩"打折"成 q 值，控制假阳性率。报 q 值是严谨性的标配（你 lessons §6 的精神）。

### 5.3 最重要的认知：「有没有结构」≠「分几型」
W5 铁证：**k=2 到 k=6 全部显著高于随机**（结构在每个分辨率都真实存在）。
但这**回答不了"到底分几个亚型"**——因为各 k 都显著。
- "有没有结构" → **null 显著性**能答（我们答了：有）。
- "分几型才对" → 是**粒度/分辨率**问题，必须用**外部标准**定：哪个分法的亚型**生存差异最大**、或**能在另一个病人队列里复现**。
这就是 W6 要做的，也是真正的**临床价值与论文新颖性**所在。

### 5.4 一个生物学线索：为什么 Her2 型没被单独分出来？
k=5 时我们的无监督簇**没干净分出 Her2 型**（LumA 反而裂成两簇）。原因很生物学：
**Her2-enriched 的本质是 HER2 基因的 DNA 扩增（拷贝数 CNV）**，它在**基因表达**层的信号不如在**拷贝数**层清楚。
→ 这正是**多组学融合**的动机：单看表达会漏掉某些亚型，加上 **CNV/甲基化** 才能补齐。（W6 评估。）

### 5.5 预告：生存分析 (survival analysis) 是亚型的"终极考官"
一个亚型有没有**临床意义**，最硬的检验是：不同亚型的**病人生存曲线是否不同**。
- **Kaplan-Meier 曲线**：随时间下降的生存概率曲线，每个亚型一条。
- **log-rank 检验**：比较多条生存曲线是否有显著差异。
W6 会用临床表里的生存数据(OS + 时间)给各 k 的亚型做这个检验——**能把病人按预后分开的分法，才是有用的分法**。

## 第 6 课 — 生存分析实战 + 一个关键负结果（紧扣 W6）

### 6.1 总生存(OS) 与"删失(censoring)"
- **总生存 OS**：从确诊到**死亡**的时间。每个病人有两个量：**时间** + **事件**(1=死亡, 0=删失)。
- **删失 (censoring)**：研究结束时病人还活着（或失访）——我们只知道"至少活了 t 年"，不知确切死亡时间。
  CS 类比：**右删失 = 观测被截断的不完整标签**。KM/log-rank 专门为处理删失而生（不能简单当缺失值丢掉）。
- 我们的构造：死亡→time=确诊到死亡天数；存活→time=末次随访天数(删失)。队列 834 例、127 死亡、随访中位 2.7 年。

### 6.2 log-rank 检验 = 比较多条生存曲线
逐个"死亡时刻"比较"各亚型实际死亡数 vs 按风险人数期望死亡数"，累积成一个卡方统计量(自由度=组数−1)。
p 小 = 各亚型生存曲线确有差异。我用 numpy+scipy 自实现（lifelines 没装）。

### 6.3 关键负结果：我们的亚型"稳定却不预后"
| 分型 | log-rank p |
|---|---|
| 我们的无监督亚型(k=2..6) | 0.19 ~ 0.30（**都不显著**）|
| PAM50(临床金标准) | **0.014（显著）** |

**我们稳定的亚型分不开生存，PAM50 能。** 这是高质量的诚实负结果，含义深刻：
- **稳定 ≠ 临床有用**：W5 证明我们的亚型很稳，W6 证明它们没预后价值。两者是两回事。
- **最易变的基因 ≠ 最预后的基因**：无监督取"方差最大"的基因，抓到的是**ER/Basal 这条最大变异轴**——它稳定、真实，但不是最能预测生死的划分。
- **PAM50 为什么行**：它是**有监督、专为预后/生物学精选**的 50 基因签名。无监督不知道"预后"这个目标，自然不会朝它优化。

### 6.4 为什么负结果是好事（科研观）
它**早早证伪了"纯无监督就够"的天真路径**，省下在错路上的时间。真正的临床价值需要让方法**知道"预后"这个目标**
（预后感知的特征选择/表示），或引入**多组学**补信号。这也把项目的论文骨架从"发现更好的亚型"（打不过 PAM50）
转向更诚实、也更有方法学价值的**"亚型认证框架"**：一套同时检验 **稳定性 + 预后 + 跨队列复现** 的标准，
并揭示"很多稳定的亚型其实不预后"。下一步 W7 选哪条，是 PM 决策。

## 第 7 课 — 循环论证陷阱 / 特征选择过拟合 / 留出验证（紧扣 W7）

### 7.1 循环论证 / 双重浸染 (circularity / double-dipping)
W7 想让聚类"知道预后"：按生存挑基因再聚类。**但若在同一批数据上"挑→聚→测"，就是循环论证**——
你专门挑了"与生存相关"的基因，它当然能分开生存。这等于**在测试集上调参/数据泄漏**（你 lessons §6 的大忌）。
**实证**：预后感知在 **训练集 p≈0.001**（看着完美）vs **留出测试集 p≈0.234**（真相）。这个巨大落差就是陷阱的铁证。
CS 类比：拿训练集准确率当模型水平——必然虚高。

### 7.2 特征选择的多重比较过拟合
我们在 ~75 个死亡事件上筛了 2000 个基因。**基因那么多、事件那么少，总有些基因"碰巧"和生存相关**（假阳性）。
这些"预后基因"换一批病人就失效——所以预后感知在留出集上**没跑赢方差基线**。
教训：**特征数 ≫ 样本/事件数时，按结局挑特征极易过拟合**，必须留出验证才能知道是真信号还是噪声。

### 7.3 留出验证 = 不可省的诚实底线
正确做法（W7 已实现）：**患者级 train/test 划分** → 训练集挑基因+建亚型 → **留出测试集**才测生存。
留出集上的 p 才是诚实的。这正是把项目做成"认证框架(A)"的价值：**自动揪出循环论证的伪预后签名**——
文献里这类错误极多，一个能识破它的套件本身就有发表价值。

### 7.4 统计功效 (power) 与样本量
测试集只有 ~334 例、~50 死亡 → **功效低**，p 值噪声大（连金标准 PAM50 在小测试集也只 40% 显著，
而全队列 p=0.014）。这解释了为何要：**更大队列 / 多次划分 / 更敏感的终点(DSS/PFI) / 多组学**。

### 7.5 为什么转向多组学 (W8)
表达层的稳健预后信号有限。**HER2 扩增（拷贝数 CNV）、基因组不稳定**等强预后信号在 **CNV/甲基化** 层可能更清楚、更稳。
W8 引入多组学，既为抢救"预后感知发现(B)"，也顺带可能补上 W5 没分出的 Her2 型。
