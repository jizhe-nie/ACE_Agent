\begin{abstract}
本文档是 ACE Agent 的核心技术指南。系统采用 Orchestrator-Worker 架构，通过语义路由分发任务，并依靠专家智能体的自愈循环（Self-healing Loop）在隔离沙箱内生成并执行数据科学代码。本版本（v2.0）特别强化了对 RAG 知识检索、意图动态判定以及专家算法职责域的实现细节描述。
\end{abstract}

\section{核心架构与逻辑平面}

\subsection{语义路由 (MasterRouter) 的判定逻辑}
\texttt{MasterRouter} 是系统的指挥中心，位于 \texttt{agent\_core/router.py}。它不依赖传统正则表达式，而是通过 LLM 语义分析将输入映射为 JSON 格式的意图对象。

\paragraph{意图分类逻辑}
\begin{itemize}
    \item \textbf{NEW\_TASK}: 当用户输入包含“分析”、“运行”、“聚类”或具体算法名（如“谱聚类”）时触发。
    \item \textbf{FOLLOW\_UP}: 当输入包含“解释”、“为什么”、“可视化”或针对已有结果的追问时触发。
\end{itemize}

\paragraph{函数实现细节：\texttt{analyze\_intent}}
系统将最近 3 轮对话历史 (\texttt{history}) 压缩后连同当前 Prompt 送入 LLM，并使用正则 \texttt{re.search(r'\{.*\}', ...)} 强制提取 JSON 块。

\subsection{主控编排器 (ACESupervisor) 的调度算法}
\texttt{ACESupervisor} 负责协调知识引擎、路由与专家代理。

\paragraph{核心调度流 (\texttt{run} 方法)}
\begin{enumerate}
    \item \textbf{意图捕获}: 调用 \texttt{MasterRouter}。
    \item \textbf{知识注入 (RAG)}: 调用 \texttt{KnowledgeEngine.query}，若匹配到学术背景，则将知识文本前置于用户 Prompt。
    \item \textbf{专家激活}: 
        \begin{itemize}
            \item 如果是 \texttt{FOLLOW\_UP}，直接进入 LLM 回答逻辑。
            \item 如果是 \texttt{NEW\_TASK}，并发激活 \texttt{CentroidExpert} 和 \texttt{TopologyExpert}。
        \end{itemize}
    \item \textbf{评分与摘要}: 对所有算法结果按 \texttt{metrics['score']} 降序排列，取首位作为优胜算法生成摘要。
\end{enumerate}

\section{专家智能体实现细节 (Worker Implementation)}

\subsection{专家职责域划分}
系统将算法按数学逻辑划分为两个主要专家：
\begin{itemize}
    \item \textbf{质心专家 (CentroidExpert)}: 负责 KMeans, GMM, MiniBatchKMeans。
    \item \textbf{拓扑专家 (TopologyExpert)}: 负责 DBSCAN, SpectralClustering (谱聚类), Agglomerative, OPTICS。
\end{itemize}

\subsection{自愈循环 (Self-Correction Loop) 实现}
实现在 \texttt{expert\_sub\_agents/base.py} 的 \texttt{execute\_with\_self\_correction} 方法中。

\paragraph{算法流程}
\begin{lstlisting}[language=Python]
for attempt in range(1, 4):
    code = generate_or_fix_code(prompt, error_msg)
    run_result = sandbox.execute(code, X, y)
    if run_result.success:
        save_artifacts()
        return results
    else:
        extract_traceback() # 提取最后 200 字符报错
\end{lstlisting}

\section{RAG 知识引擎 (Knowledge Engine)}

\subsection{知识扫描与去重}
\texttt{KnowledgeEngine} 使用 \texttt{ingested\_manifest.json} 维护一个 MD5 哈希列表。扫描 \texttt{docs/} 目录时，仅处理哈希值变动或新增的文件。

\subsection{向量检索配置}
\begin{itemize}
    \item \textbf{Embedding 模型}: \texttt{all-MiniLM-L6-v2} (本地运行)。
    \item \textbf{持久化库}: ChromaDB。
    \item \textbf{环境变量注入}: 强制设置 \texttt{HF\_HUB\_OFFLINE=1} 以确保离线环境下的启动速度。
\end{itemize}

\section{安全沙箱与代码隔离 (CoderSandbox)}

\subsection{执行环境构造}
在 \texttt{tools/coder\_sandbox.py} 中，通过 \texttt{exec()} 运行代码前会初始化一个受限字典 \texttt{exec\_env}。

\paragraph{注入变量清单}
\begin{itemize}
    \item \texttt{X, y}: 内存中的 numpy 数组。
    \item \texttt{plt, np}: 预导入的数据科学库。
    \item \texttt{artifacts}: 空字典，用于专家回传结果。
\end{itemize}

\paragraph{输出捕获机制}
专家生成的代码必须将结果存入 \texttt{artifacts['算法名']}，包含 \texttt{labels}、\texttt{metrics} 和 \texttt{plot\_path} 三个必填字段，否则会被判定为执行失败。

\section{API 参考手册 (API Reference)}

\subsection{\texttt{UniversalLLMClient}}
\begin{itemize}
    \item \texttt{chat\_completion(messages, system\_prompt)}: 具备自动重试机制的通用接口。
    \item \texttt{summarize\_report(payload)}: 接收 \texttt{user\_intent} 参数，实现意图优先的报告摘要。
\end{itemize}

\subsection{\texttt{LatexReportGenerator}}
\begin{itemize}
    \item \texttt{generate(report)}: 动态扫描 \texttt{report.results}，为每个算法生成 \texttt{\textbackslash subsection}，并将 \texttt{plot\_path} 转换为 \LaTeX 路径。
\end{itemize}

\section{部署与排烟测试}

\subsection{中文字体支持方案}
由于在 Windows 环境下 \texttt{matplotlib} 默认不识别中文，系统在启动时会检测 \texttt{Win32} 平台并尝试加载“微软雅黑”或“黑体”。

\subsection{离线模式 (Offline Mode)}
通过屏蔽 \texttt{transformers} 的日志输出 (\texttt{VERBOSITY=error})，解决了大规模知识库加载时的终端卡顿问题。
