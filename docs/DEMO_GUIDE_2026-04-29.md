## 📂 二、 演示案例库（按研究维度划分）

### 1. 非凸流形结构挑战 (Topological Robustness)

* **数据集**：`moons` (月牙), `smile` (笑脸)
* **演示重点**：展示系统在处理球形假设失效时的**算法退避 (Backoff) 逻辑**。
* **技术看点**：`TopologyExpert` 自动计算 K-距离启发式参数，并在 DBSCAN 失效时自愈切换至 HDBSCAN。

### 2. 高维表征与维度感知 (High-Dim Latent Space)

* **数据集**：`digits` (64维手写字符), `mnist` (784维图像特征)
* **演示重点**：展示**维度感知调度**与**深度学习管线**的自动化协同。
* **技术看点**：Dimension Expert 自动激活，动态构建包含 PCA、UMAP 和深度自编码器（AE）的竞争性实验链。

### 3. 系统鲁棒性与自愈 (Self-Healing Mechanism)

* **场景模拟**：使用自然语言诱导 LLM 生成包含 API 变更或逻辑错误的代码。
* **演示重点**：展示 **Think-Act-Fix** 循环如何捕捉 Traceback 并实时修复代码。
* **技术看点**：不中断分析流程，实现“运行时报错 -> 语义分析 -> 增量修复 -> 重新验证”的闭环。

3. **技术观测**：
    * 观察 **MasterRouter** 进行意图三分类（NEW_TASK）。
    * 展示 **ACESupervisor** 并行调度多个专家（Centroid, Topology, Zoo）。
    * 展示 **Critic 审计卡片**（Web UI 报告页黄色/绿色/红色提示框），逐条讲解 endorsement、Bootstrap 稳定性、Hopkins 趋势、过拟合风险评估、聚类数一致性检查等审计结论。
    * 若生成 LaTeX 报告，展示 **独立审计章节** 的排版效果。

### 场景 B：深度管线与高维性能（核心创新展示）

**汇报逻辑**：针对复杂高维数据，展示系统在深度学习应用上的突破。

1. **高维加载**：选择 `digits` (64维)。
2. **指令输入**：使用“*对此数据集进行深度嵌入聚类，对比线性与非线性降维的效果。*”

## 🚀 五、 后续研发路径 (Roadmap)

1. **DEC/IDEC 联合优化 (P0)**：将目前的”降维+聚类”两阶段训练升级为基于 KL 散度的端到端联合优化，目标将高维数据的 ARI 提升至 0.7 以上。
2. **显存自适应策略 (P1)**：针对本地设备（RTX 4060 Ti）与云端设备（A4000）自动匹配最优 Batch Size 与计算精度。
3. **约束聚类 (P2)**：引入 Human-in-the-Loop 机制，支持用户通过自然语言施加逻辑约束。
4. **KIM 知识整合 (P3)**：跨语言脚本集成（MATLAB/R），LLM 自主检索并调用外部专业算法。
5. **RAG 独立引擎 (P3)**：部署独立向量数据库或云托管 RAG 服务替代手写 knowledge_engine.py。
