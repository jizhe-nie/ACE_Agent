# 可行性研判：这个项目到底能不能冲一区，该打什么靶

> 作者：Claude（PM）· 日期：2026-06-23 · 触发：W10 创新失败后，PM 决定**暂停建模、重审可行性**。
> 原则：不灌水、不唱衰。基于 10 周实测 + 文献证据，给**校准的**结论。配套：`Review/ENGINEERING_LOG.md`、`docs/RESEARCH_DIRECTION_2026-06-22.md`。

## 0. 一句话结论

**当前靶子（在单队列、欠功效的 TCGA-BRCA 总生存上、用无监督多组学超越成熟监督标准 PAM50）几乎赢不了——
任何方法都难，不是我们的方法不行。** 要冲一区，必须**换靶**到领域公认的开放缺口上。最匹配你资源、且真有 Q1 机会的靶是
**跨队列/跨平台稳健的亚型发现（TCGA↔METABRIC, RNA-seq↔芯片）**，可叠加**不完整组学**这个公认难题。
**但要诚实：即便换对靶，这也是 3–6 个月的认真工程 + 需要把 GPU 装上 + Q1 不保证**；更现实的"地板"是一篇扎实的二区方法论。

## 1. 10 周确立的事实（别浪费这些教训）

- ✅ **W9 深度 L2 是唯一正信号**：AE+DEC 把生存 log-rank 0.83→0.11（胜 sklearn 弱基线）。说明深度表示 substrate 有效。
- ❌ 其余全负且**始终没超 PAM50**：W6 稳定≠预后、W7 预后感知循环+不泛化、W8/W10 焦点融合救不回 Her2。
- 🔑 **关键教训**：① 单队列 BRCA-OS 欠功效（123 事件）→ 任何方法都难显著；② 朴素/学习式融合会稀释驱动焦点（ERBB2/Her2）。

## 2. 领域的真实门槛（文献证据）

- **已有 2025 系统基准**（9 TCGA 癌种 × 11 组学组合）：要喊 SOTA 得正面打赢 **NEMO / iClusterBayes / Subtype-GAN / SNF / PINS / LRAcluster** 等成熟方法（silhouette/NMI ~0.89）。对个人/CPU 是**高门槛**。
- **我们的 W8 负结果是公认现象**：该基准明确"**集成更多组学不总更好**"——所以"朴素多组学更强"不是可赢命题（但佐证我们没乱来）。
- **Q1 venue 确认**：`Briefings in Bioinformatics`、`Bioinformatics`(Oxford)（中科院一区）常发此类方法/基准/综述——**目标 venue 具体且够得着**。

## 3. 领域公认的开放缺口（真空白，可打）

1. **不完整 / 部分组学**：现实中很多病人缺某些组学层，多数方法却要求"人人全组学"。鲁棒处理缺失是公认难题。
2. **跨队列 / 跨平台异质性**：TCGA(RNA-seq) ↔ METABRIC(微阵列) 的平台漂移使亚型难复现——**公认协议(train-on-source/test-on-target)、有明确指标**（外部 F1；域适应实测 0.738→0.801）。
3. **可解释性**：深度模型输出难转成生物学/临床洞见——公认瓶颈（但难定清晰的"取胜指标"）。
4. **无通用框架**：方法表现随数据类型而变，没有 one-size-fits-all。

## 4. 候选命题可行性矩阵

| 命题 | 新颖性 | 可赢性 | 资源成本 | 与你匹配 | Q1 概率(诚实) |
|---|---|---|---|---|---|
| A. 单癌种超 PAM50(现靶) | 低(已被做烂) | **极低**(欠功效) | 低 | 高 | **~5%** |
| B. 泛癌基准超深度 SOTA | 中 | 低-中(打成熟方法难) | **高**(多癌种+GPU) | 中 | ~15% |
| **C. 跨队列/平台稳健亚型(TCGA↔METABRIC)** | **中-高** | **中**(公认难点=机会) | 中(+METABRIC+GPU) | **高**(复用现有+你的严谨) | **~25–35%** |
| D. 不完整组学鲁棒整合 | 中-高 | 中 | 中-高 | 中 | ~20% |
| **C+D 合体** | **高** | 中 | 中-高 | 高 | **~30%** + 更完整故事 |
| E. 认证/基准论文(无新方法) | 低-中 | 高(易发) | 低 | 高 | 二区为主(PM 已否) |

> 概率是**粗校准的主观估计**，用于相对排序，非承诺。绝对值都不高——这是诚实的：个人从零超成熟生信 SOTA 本就难。

## 5. 资源现实校准（必须面对）

- **你是个人、CS 背景、正在学生物、当前 torch 是 CPU 版（GPU 待装）、只有公开数据、无领域协作者。**
- 现实里，这类一区论文通常出自**有领域专家 + GPU + 数月专注**的团队。个人单挑成熟 SOTA 的成功案例**少**。
- **不是劝退**，是定预期：把"地板"设为一篇**扎实的二区方法论**（可达），把一区当**够一够的 stretch**（需 C/C+D 打对 + 严谨验证 + 可能需要一点领域输入/协作）。

## 6. 诚实结论与推荐

**推荐主攻 C（跨队列稳健亚型），有余力叠 D（不完整组学）。** 理由：
- 打在**公认开放缺口**上（不是和 PAM50 死磕单队列 OS）。
- **取胜指标清晰且可赢**：亚型在 TCGA↔METABRIC 间的**可复现性/迁移**（外部一致性、跨平台稳定），而非欠功效的单队列 OS。
- **复用我们已有资产**：W9 深度内核 + 严谨评估（留出/null 校准/多 seed）正好是跨队列验证需要的。
- **平台漂移(RNA-seq vs 芯片)是真难点** → 一个稳健方法就是真贡献，别人能学能用。

**先决条件（缺一不可）**：① 把 **CUDA torch 装上**（GPU）；② 下 **METABRIC**（cBioPortal，~2000 例，表达+CNV+临床+PAM50）。

## 7. 若推进：先做"可行性验证"再投入（避免又烧几周）

不一上来就建大模型。先用**最小实验**验证 C 这个靶**值不值得做**（1 个脚本即可）：
1. 取 TCGA-BRCA 与 METABRIC 的**共同基因**，对齐 PAM50 标签。
2. 量化**平台漂移有多严重**：在 TCGA 训练一个简单亚型分类器，测 METABRIC 外部一致性（baseline）。
3. 看**现有简单方法的跨队列差距**有多大——差距大 = 有提升空间 = 靶值得打；差距小 = 已被解决，换靶。
→ 这一步**几天、可在 CPU 跑**，用结果决定是否全力投入 C。**先验证靶，再建方法。** 这正是我们之前跳过、导致绕弯的一步。
