## 第 1 课 — 多组学 / PAM50 / 批次效应（理解亚型体检的最小知识）

### 1.1 为什么"乳腺癌"要分亚型
"乳腺癌"不是一种病，而是分子层面差异巨大的一组病：有的激素治疗有效、预后好；有的（三阴性）凶险无靶向药。
**分子亚型 = 把"一种病"拆成治疗/预后不同的子群**。"发现亚型"的临床意义即在此，也是论文价值锚。

### 1.2 多组学（multi-omics）—— 用 CS 类比
中心法则：**DNA →(转录)→ RNA →(翻译)→ 蛋白质**。每层可测量 = 一个"组学层"。同一批病人多层测量 = **多模态学习**。

| 组学层 | 测什么 | CS 类比 | 矩阵形态 |
|---|---|---|---|
| 基因表达 (RNA-seq) | 每个基因此刻多活跃 | 哪些代码模块在跑、跑多频 | 样本×基因(~2万)，值=表达量 |
| DNA 甲基化 | 表观"开关"，把基因调静音 | feature flag / 注释掉代码 | 样本×CpG 位点 |
| 拷贝数变异 (CNV) | 某基因区被扩增/缺失几份 | 某模块被复制/删除 | 样本×基因区段 |

直觉：**DNA 在每个细胞基本相同（源码一样），但"哪些基因在跑、跑多猛"（表达）才区分亚型** → 表达层信息量最大，先用它。

### 1.3 PAM50 —— 我们的"金标准标签"（验证锚）
用 50 个基因把乳腺癌分成 **5 个公认亚型**的成熟方法。它是临床/文献承认的标准，我们拿它当**弱标签**验证聚类是否抓到真生物学（算 ARI/NMI）。

| PAM50 亚型 | 粗略含义 |
|---|---|
| Luminal A / B | 激素受体阳性；A 预后较好、B 较差 |
| HER2-enriched | HER2 基因驱动，有靶向药 |
| Basal-like | 多为三阴性，侵袭性强 |
| Normal-like | 接近正常组织 |

注意：PAM50 本身也是一种聚类结论、不是绝对真理。我们**不复刻它**，而是用它做合理性检验；创新在于更稳、更可解释、甚至它没覆盖的新亚型。

### 1.4 批次效应（batch effect）= 生物学版"域偏移"
TCGA 样本来自几十家医院（**TSS, tissue source site**）、不同测序板、不同试剂批次，各留**技术指纹**。
CS 类比：**dataset/domain shift**（像模型学会识别水印而非内容）。
**危险**：不处理则聚类会**按医院/批次分群而非生物学**。体检必查："簇是否与批次强相关？"

## 第 2 课 — 读懂真实 TCGA-BRCA 体检结果（紧扣 W1 数字）

体检结论：956 样本，Hopkins 0.694，ARI(KMeans,PAM50)=0.434 ≫ ARI(KMeans,TSS)=0.024 → **绿灯**。

### 2.1 为什么 262 个样本没 PAM50 标签、要剔除
TCGA 里除肿瘤外还有**癌旁正常组织**（条码 `-11`）和少数未分型样本，它们无 PAM50 调用。
真实组学数据天然"脏"——**先过滤到有金标准的队列再比较**是基本功。

### 2.2 PAM50 五型为何不均衡（LumA 434 ≫ Her2 67）
真实临床分布：Luminal A（激素受体阳性、预后最好）最常见；Her2-enriched、Basal（多三阴性、侵袭强）较少见。
**含义**：罕见亚型样本少、最难被聚类抓到 → 这是 **IP-4（开集/罕见新亚型发现）** 的机会；也说明**不能只看整体 ARI**（会被大类 LumA 主导）。

### 2.3 "ARI=0.434" 怎么用
这是**朴素 KMeans 探针**的成绩 = **基线地板（baseline to beat）**。目标**不是刷 ARI 到 1.0 复刻 PAM50**（无新意），
而是产出**更稳、更可解释、甚至 PAM50 没覆盖的亚型**。ARI 只是"没跑偏"的下限。

### 2.4 科学家的克制
簇-批次 ARI 低 ≠ 绝对无批次效应，簇**内部**仍可能有细微偏移。所以后续仍把 `tissue_source_site` 当协变量监控，而非从此无视。

## 第 3 课 — 多原型 / medoid / 质心陷阱（紧扣 W3 结果，含一个负结果与一个数据坑）

### 3.1 质心 vs medoid（核心概念）
- **质心 (centroid)**：一簇所有点的**平均**。问题：对长条/S形/流形簇，平均点可能落在**样本不存在的空白处**，
  不代表任何真实病人（lessons §5 "质心陷阱"）。高维组学更易中招。
- **medoid**：簇内**最居中的真实样本**（到簇内其它点总距离最小）。永不落空白 → 是一位**真实病人范例**，天然可解释。
- **多原型 (multi-prototype)**：一个亚型用**多个** medoid 代表（而非 1 个中心），以刻画复杂形状。
  CS 类比：与其用"一个平均向量"概括一类，不如用"几个真实代表样本"——像用几个典型用例描述一个聚类。

### 3.2 W3 的诚实负结果（重要）
我们在 BRCA 表达上比了 4 法，**多原型(IP-3) ARI 0.404 < KMeans 0.434 ≈ K-medoid 0.439**，没赢。
**这不是失败，是真信息**：说明 **BRCA 表达的亚型在降维空间相对"凸/团块"**，质心陷阱在这儿不严重 →
多原型的优势没机会发挥，反而其额外步骤带来噪声（方差更大）。
**教训（呼应你 lessons §7 的"对抗去域不是免费午餐"）**：**别假设一个高级方法一定更好，必须实测**。
→ 战略微调：对 BRCA 表达，**IP-1（稳定性）比 IP-3 更对症**；IP-3 可能要到**多组学融合**或非凸模态才显威力。

### 3.3 一个真实数据坑：`-11` 癌旁正常组织
聚类暴露：某个亚型的范例病人条码以 **`-11`** 结尾。TCGA 条码第 4 段是**样本类型码**：
**`-01` = 原发肿瘤**，**`-11` = 癌旁正常组织**（normal adjacent）。
PAM50 的 "Normal" 类**混入了真正的正常组织样本**。做**纯肿瘤亚型**时通常要**剔除 `-11`**——
否则你的"一个亚型"其实是"正常组织 vs 肿瘤"的平凡区分，而非肿瘤内部分型。下一步 W4 会剔除它重跑。

### 3.4 为什么要"多 seed + 子采样稳定性"
KMeans 等有随机初始化 → **单跑数字不可信**（lessons §8）。所以报 5 个种子的 **mean±std**。
更进一步，**子采样稳定性**（抽 80% 重聚类，看与全量结果是否一致，我们测得 0.474）是 **IP-1** 的雏形：
一个"真亚型"应当在**换一批病人/重采样下依然稳定**，不稳的多半是过拟合的伪亚型。

## 第 4 课 — 共识聚类的可复现性危机 / 零模型 / 乳腺癌的主分子轴（紧扣 W4）

### 4.1 为什么剔除正常组织后 ARI 反而下降（0.434 → 0.337）？
我们剔除了 `-11` 癌旁正常组织（PAM50 "Normal" 类 119→24，证实大多是正常组织）。结果 ARI 掉了——**这是好事、是诚实**：
- "肿瘤 vs 正常组织"差异巨大、**极易分**，它的存在把整体 ARI **虚高**了。
- 剔除后只剩"肿瘤内部亚型彼此区分"，本就**更微妙更难**。
**教训**：警惕被"平凡的易分项"抬高的指标——真正难、也真正有价值的是亚型间的细分。

### 4.2 共识聚类的"可复现性危机"（本项目要攻的痛点）
癌症亚型主流工具 **consensus clustering / ConsensusClusterPlus**（重采样多次、看哪些样本总聚在一起）被一篇著名论文
（**Șenbabaoğlu et al. 2014**）证明**过度乐观**：**即使数据没有真实结构，它也会"发现"看似稳定的簇并背书**。
根因：它**不和"零模型"比较**——只问"重采样下稳不稳"，不问"随机数据是不是也这么稳"。

### 4.3 零模型（null model）= 给稳定性一把"随机标尺"
**零模型**：把每个基因的数值在样本间**独立打乱**——保留每个基因自己的分布，但**摧毁基因之间的联合结构**（也就是簇结构）。
CS 类比：把表格每一列各自洗牌，行与行之间的"模式"就没了。
我们的 W4 结果：真实数据稳定性 **0.99**，零模型只有 **0.02**（贴地）。**真 ≫ null → 结构是真的**，不是 consensus clustering 那种自欺。这就是 **IP-1 的核心机制**。

### 4.4 k=2 是什么？乳腺癌的"主分子轴"
我们的方法（暂用 naive 选 k）认证出 **k\*=2**，且一簇由 **Basal** 主导。这在生物学上完全正确：
乳腺癌最主要的一刀是 **ER 阴性 / Basal-like（≈三阴性，侵袭强）** vs **ER 阳性 / Luminal**。
所以方法找到了**最稳健的那条真实生物学分界**。

### 4.5 但"按最大稳定性选 k"有偏置 → 下一步怎么改
稳定性曲线随 k **单调下降**：**越粗的划分天生越可复现**（k=2 永远最稳）。所以**不能简单按最大稳定性选 k**，
否则永远只给你 2 类、给不出临床在用的 5 类。
**W5 的更优思路**：不逼出单一 k，而是输出**"认证的亚型层级"**——k=2 高置信、k=5 仍显著高于 null、k=7 渐弱，
**逐层报告置信度**。这直接回应"亚型数到底取几"这个长期任意、没标准答案的痛点，比硬选一个 k 更诚实、也更有论文创新点。

## 第 5 课 — 置换检验 / p 值 / 多重检验 / "有没有结构"≠"分几型"（紧扣 W5）

### 5.1 置换检验 (permutation test) = 用"洗牌"造一把显著性尺子
W4 只比了一次 null；W5 把 null **做 R=20 次**（每次重新洗牌→重新算稳定性），得到稳定性的**零分布**。
- **经验 p 值** = 「随机数据稳定性 ≥ 真实稳定性」的比例。CS 类比：蒙特卡洛——洗很多次牌，看真实结果有多"出格"。
- **z 分数** = (真实 − null均值) / null标准差。我们测到 **z=89~239**（真实比随机高出近百个标准差）→ 结构极其确凿。
- 注意：p 最小只能到 1/(R+1)。R=20 → p 下限 0.048，所以我们所有 k 的 p 都"卡"在 0.048；**真正的强弱看 z**。正式论文会用 R≥1000。

### 5.2 多重检验校正 (BH/FDR) = 同时问很多问题要"打折"
我们对 5 个 k 各做一次检验。**做的检验越多，纯靠运气出现"显著"的概率越大**（CS 类比：多重比较/p-hacking）。
**Benjamini-Hochberg (BH)** 把 p 值按秩"打折"成 q 值，控制假阳性率。报 q 值是严谨性的标配（你 lessons §6 的精神）。

### 5.3 最重要的认知：「有没有结构」≠「分几型」
W5 铁证：**k=2 到 k=6 全部显著高于随机**（结构在每个分辨率都真实存在）。
但这**回答不了"到底分几个亚型"**——因为各 k 都显著。
- "有没有结构" → **null 显著性**能答（我们答了：有）。
- "分几型才对" → 是**粒度/分辨率**问题，必须用**外部标准**定：哪个分法的亚型**生存差异最大**、或**能在另一个病人队列里复现**。
这就是 W6 要做的，也是真正的**临床价值与论文新颖性**所在。

### 5.4 一个生物学线索：为什么 Her2 型没被单独分出来？
k=5 时我们的无监督簇**没干净分出 Her2 型**（LumA 反而裂成两簇）。原因很生物学：
**Her2-enriched 的本质是 HER2 基因的 DNA 扩增（拷贝数 CNV）**，它在**基因表达**层的信号不如在**拷贝数**层清楚。
→ 这正是**多组学融合**的动机：单看表达会漏掉某些亚型，加上 **CNV/甲基化** 才能补齐。（W6 评估。）

### 5.5 预告：生存分析 (survival analysis) 是亚型的"终极考官"
一个亚型有没有**临床意义**，最硬的检验是：不同亚型的**病人生存曲线是否不同**。
- **Kaplan-Meier 曲线**：随时间下降的生存概率曲线，每个亚型一条。
- **log-rank 检验**：比较多条生存曲线是否有显著差异。
W6 会用临床表里的生存数据(OS + 时间)给各 k 的亚型做这个检验——**能把病人按预后分开的分法，才是有用的分法**。

## 第 6 课 — 生存分析实战 + 一个关键负结果（紧扣 W6）

### 6.1 总生存(OS) 与"删失(censoring)"
- **总生存 OS**：从确诊到**死亡**的时间。每个病人有两个量：**时间** + **事件**(1=死亡, 0=删失)。
- **删失 (censoring)**：研究结束时病人还活着（或失访）——我们只知道"至少活了 t 年"，不知确切死亡时间。
  CS 类比：**右删失 = 观测被截断的不完整标签**。KM/log-rank 专门为处理删失而生（不能简单当缺失值丢掉）。
- 我们的构造：死亡→time=确诊到死亡天数；存活→time=末次随访天数(删失)。队列 834 例、127 死亡、随访中位 2.7 年。

### 6.2 log-rank 检验 = 比较多条生存曲线
逐个"死亡时刻"比较"各亚型实际死亡数 vs 按风险人数期望死亡数"，累积成一个卡方统计量(自由度=组数−1)。
p 小 = 各亚型生存曲线确有差异。我用 numpy+scipy 自实现（lifelines 没装）。

### 6.3 关键负结果：我们的亚型"稳定却不预后"
| 分型 | log-rank p |
|---|---|
| 我们的无监督亚型(k=2..6) | 0.19 ~ 0.30（**都不显著**）|
| PAM50(临床金标准) | **0.014（显著）** |

**我们稳定的亚型分不开生存，PAM50 能。** 这是高质量的诚实负结果，含义深刻：
- **稳定 ≠ 临床有用**：W5 证明我们的亚型很稳，W6 证明它们没预后价值。两者是两回事。
- **最易变的基因 ≠ 最预后的基因**：无监督取"方差最大"的基因，抓到的是**ER/Basal 这条最大变异轴**——它稳定、真实，但不是最能预测生死的划分。
- **PAM50 为什么行**：它是**有监督、专为预后/生物学精选**的 50 基因签名。无监督不知道"预后"这个目标，自然不会朝它优化。

### 6.4 为什么负结果是好事（科研观）
它**早早证伪了"纯无监督就够"的天真路径**，省下在错路上的时间。真正的临床价值需要让方法**知道"预后"这个目标**
（预后感知的特征选择/表示），或引入**多组学**补信号。这也把项目的论文骨架从"发现更好的亚型"（打不过 PAM50）
转向更诚实、也更有方法学价值的**"亚型认证框架"**：一套同时检验 **稳定性 + 预后 + 跨队列复现** 的标准，
并揭示"很多稳定的亚型其实不预后"。下一步 W7 选哪条，是 PM 决策。

## 第 7 课 — 循环论证陷阱 / 特征选择过拟合 / 留出验证（紧扣 W7）

### 7.1 循环论证 / 双重浸染 (circularity / double-dipping)
W7 想让聚类"知道预后"：按生存挑基因再聚类。**但若在同一批数据上"挑→聚→测"，就是循环论证**——
你专门挑了"与生存相关"的基因，它当然能分开生存。这等于**在测试集上调参/数据泄漏**（你 lessons §6 的大忌）。
**实证**：预后感知在 **训练集 p≈0.001**（看着完美）vs **留出测试集 p≈0.234**（真相）。这个巨大落差就是陷阱的铁证。
CS 类比：拿训练集准确率当模型水平——必然虚高。

### 7.2 特征选择的多重比较过拟合
我们在 ~75 个死亡事件上筛了 2000 个基因。**基因那么多、事件那么少，总有些基因"碰巧"和生存相关**（假阳性）。
这些"预后基因"换一批病人就失效——所以预后感知在留出集上**没跑赢方差基线**。
教训：**特征数 ≫ 样本/事件数时，按结局挑特征极易过拟合**，必须留出验证才能知道是真信号还是噪声。

### 7.3 留出验证 = 不可省的诚实底线
正确做法（W7 已实现）：**患者级 train/test 划分** → 训练集挑基因+建亚型 → **留出测试集**才测生存。
留出集上的 p 才是诚实的。这正是把项目做成"认证框架(A)"的价值：**自动揪出循环论证的伪预后签名**——
文献里这类错误极多，一个能识破它的套件本身就有发表价值。

### 7.4 统计功效 (power) 与样本量
测试集只有 ~334 例、~50 死亡 → **功效低**，p 值噪声大（连金标准 PAM50 在小测试集也只 40% 显著，
而全队列 p=0.014）。这解释了为何要：**更大队列 / 多次划分 / 更敏感的终点(DSS/PFI) / 多组学**。

### 7.5 为什么转向多组学 (W8)
表达层的稳健预后信号有限。**HER2 扩增（拷贝数 CNV）、基因组不稳定**等强预后信号在 **CNV/甲基化** 层可能更清楚、更稳。
W8 引入多组学，既为抢救"预后感知发现(B)"，也顺带可能补上 W5 没分出的 Her2 型。

## 第 8 课 — 拷贝数(CNV) / 多组学融合 / "焦点信号被稀释"（紧扣 W8）

### 8.1 拷贝数变异 (CNV) 与 GISTIC2
**CNV**：某段 DNA 被**扩增(多拷贝)** 或 **缺失**。癌症常见整条染色体臂的增减(臂级)，也有针对单个癌基因的**焦点扩增**。
**GISTIC2** 是把原始拷贝数处理成"每个基因的拷贝数估计值"(可正可负，0=正常二倍体)的标准工具。我们下的就是它的基因级矩阵。

### 8.2 生物学铁证：Her2 = ERBB2 焦点扩增
我们直接看 **ERBB2(=HER2) 基因的拷贝数**，按 PAM50 分组：**Her2 型 = +2.60**，其余 ≈0。
这是教科书级印证：**Her2 亚型的定义性事件就是 ERBB2 基因的焦点扩增**——CNV 层清清楚楚带着这个信号。

### 8.3 多组学早融合 (early integration)
**早融合**：把多个组学的特征**拼在一起**再聚类（我们做法：表达、CNV 各自 选高变→PCA→z-score→拼接）。
直觉上"信息更多应该更好"。但——

### 8.4 反直觉的负结果：朴素融合**稀释**了焦点信号
加了 CNV 后，Her2 簇召回反而从 **66% 掉到 33%**！原因很深刻：
- ERBB2 只是 2000 个 CNV 基因里的**一个**。CNV 的"方差大头"是**整条臂的增减/基因组不稳定**(广谱)，
  做 PCA 取 30 个主成分时，抓到的是这些广谱模式，**单基因 ERBB2 的焦点信号被淹没**。
- 反而**表达层**因为 ERBB2 扩增会带动**该区域基因共表达**，更能抓住 Her2。
**教训**：多组学不是"拼起来就更好"。**等权拼接会让"焦点信号"被"广谱方差"淹没**；
要用更聪明的融合（模态加权、保留焦点特征、相似网络融合 SNF），或干脆把关键基因(如 ERBB2)**显式**加进去。
CS 类比：把两个尺度/信息密度不同的特征集直接 concat，强势但无关的那组会主导距离度量。

### 8.5 大局观：负结果如何拼成一篇论文
到此发现轨道(B) 连吃三个诚实负结果：①稳定≠预后(W6) ②预后感知循环+不泛化(W7) ③朴素融合稀释+无改善(W8)。
**这不是失败——这是一篇"亚型认证框架(A)"论文的核心素材**：一套能系统揭穿"稳定假象/循环论证/融合稀释"等
常见伪发现陷阱的标准方法。很多已发表的亚型/签名正栽在这些坑里。下一步把它形式化 + 补跨队列复现 = 可投一区方法学。

> **更正（第 8 课结尾的方向被 PM 否决）**：把负结果包装成"认证框架"是美化失败。真正的问题是**我们一直没建
> 深度模型(L2)**，只跑了 sklearn 弱基线。第 9 课起转向真正的深度方法。

## 第 9 课 — 深度表示学习 / 自编码器 / DEC / ARI≠NMI（紧扣 W9，第一次见真信号）

### 9.1 自编码器 (autoencoder, AE) = 学一个"压缩表示"
AE：编码器把高维输入压成低维**潜向量 z**，解码器再从 z 重建输入；训练目标=重建误差最小。
CS 类比：**有损压缩 / 表示学习**——z 被迫保留数据最本质的结构。多组学 AE = 表达和 CNV 各自编码→融合成一个 z→各自重建。
这就是 lessons §3 的 **L2**：**让"表示"被学出来**，而不是用现成的 PCA。

### 9.2 DEC (Deep Embedded Clustering) = 表示与簇一起学
普通做法是"先 AE 再 KMeans"(表示和聚类两张皮)。**DEC** 让二者**联合优化**：用一个软分配 q(学生 t 分布)+
自我强化的目标分布 p，边调编码器边调簇心。CS 类比：**表示和聚类互相塑形**——这才是"真内核"(L2)。

### 9.3 ARI ≠ NMI：两个指标会打架
W9 里深度方法 **NMI 升(0.26→0.36) 但 ARI 略降(0.26→0.24)**。
- **ARI**：看与 PAM50 划分的**精确配对一致性**（每对样本是否同组判断一致）。
- **NMI**：看两套划分的**信息重叠**（互信息），对簇大小不均、簇数不同更宽容。
深度簇更均衡、信息上更贴 PAM50，但不在"精确配对"上对齐 → NMI↑ 而 ARI↓。**单看一个指标会误判，要并看。**

### 9.4 第一次真信号：深度让"预后"动了
8 周来生存 p 一直在 0.2~0.8（分不开）。W9 深度 L2 第一次把它压到 **0.106**（弱基线是 0.831，PAM50 是 0.009）。
**含义**：深度融合 expr+CNV 学到了**预后相关的轴**(可能对应增殖/肿瘤分级)，是 sklearn 抓不到的。
这是"建对了方法就见效"的证据——但**还没赢 PAM50，不吹**。

### 9.5 反复出现的真问题：焦点信号被稀释
Her2 召回在 W8(朴素融合) 和 W9(深度融合) **都下降**(0.75→0.43~0.50)。融合越狠，ERBB2 这种**单基因驱动焦点**越被
广谱信号淹没。→ W10 的创新就打这个点：**焦点信号保留的深度融合**——既学预后表示，又不丢驱动事件。
这是个有据、可测、别人能用的真创新（不是验证别人）。

## 第 10 课 — 焦点 vs 臂级 CNV / GISTIC 阈值 / 为何 W10 创新失败（不粉饰）

### 10.1 焦点(focal) vs 臂级(arm-level) 拷贝数
- **臂级**：一整条染色体臂的增减，影响**很多基因、很多病人**，但幅度温和（GISTIC 值 ~±1）。多是"背景"。
- **焦点(focal)**：只针对一小段（甚至单基因）的**高幅**扩增/缺失，常是**驱动事件**（如 ERBB2 扩增=Her2、MYC 扩增）。
**GISTIC 高水平阈值**：`|值|≥2` 通常表示**高幅(高拷贝)** 事件 = 焦点驱动；`≥1` 只是"有改变"(含大量臂级背景)。

### 10.2 我 W10 的 bug：用错阈值=选错基因
我最初按"`|CNV|≥1` 的样本占比"选焦点基因——结果选中的全是**臂级广谱**基因（改变频繁但不驱动），
**ERBB2 反排到 #1826**（它只在 ~15% 的 Her2 病人里高幅扩增，按"频率"吃亏）。改用 `|CNV|≥2`（高幅）后 ERBB2 升到 #16。
**教训**：生物学阈值的含义要搞对，否则"焦点通路"里装的根本不是焦点。

### 10.3 但修对后机制仍失败——为什么（诚实）
即便 ERBB2 进了焦点集，FP-DEC 仍没救回 Her2（0.495→0.448）。原因：
1. 把 200 个焦点基因**压成 16 维**，ERBB2 在焦点通路里**又被二次稀释**；
2. **Her2 只占 8%**（64/817），而 DEC 偏好**均衡簇**，16 维焦点只占聚类空间 20%，压不过主导的广谱表示。
→ "学习式压缩焦点通路"这条设计**行不通**。要救小的、驱动定义的亚型，得用更强的针对性（如把 ERBB2 等少数驱动**直接**作聚类特征，或半监督），而非压缩。

### 10.4 大局诚实话：可能是靶子错了，不是机制差
10 周下来唯一正信号是 W9（深度让生存 0.83→0.11）。其余全负。**在单队列、欠功效的 TCGA-BRCA 总生存上去超成熟标准 PAM50，对任何方法都极难**——深度多组学方法的 SOTA 优势通常在**泛癌基准**或**跨队列复现**上才显现。下一步该重选战场，而不是再调一个机制。这条要 PM 拍板。

## 第 11 课 — 跨队列验证 / TCGA vs METABRIC / "能复现才算真亚型"（紧扣可行性研判）

### 11.1 为什么"能在别的队列复现"才算真亚型
一个亚型若只在你这一批病人里成立、换一批就没了，那多半是**过拟合该队列的噪声**（呼应 lessons §6 的泄漏/过拟合）。
**金标准检验**：在**完全独立的另一个病人队列**里，同样的亚型还在不在、还分不分得开预后。能跨队列复现 = 真信号。
这就是审稿人最认的"**外部验证 (external validation)**"。

### 11.2 TCGA vs METABRIC：两个独立乳腺癌队列，但"测量仪器"不同
- **TCGA-BRCA**：~1000 例，表达用 **RNA-seq**（测序）。
- **METABRIC**：~2000 例，表达用 **微阵列 (microarray)**（芯片）。都带 PAM50 与生存。
- **关键难点**：RNA-seq 和芯片是**两种不同的测量技术**，数值尺度/分布不同——这是**批次效应的"豪华升级版"**（第 1 课的平台漂移，跨到了不同仪器+不同人群）。
  CS 类比：训练集和测试集来自**两台不同传感器**，直接迁移会崩。**一个能扛住这种平台漂移的方法，本身就是真贡献。**

### 11.3 不完整组学 (incomplete/partial multi-omics)
现实中很多病人**缺某些组学层**（没做甲基化、或 CNV 缺失）。多数方法却要求"人人齐全组学"，只能丢弃缺失病人=浪费数据。
**能优雅处理缺失组学**是公认开放难题——也是个可打的真缺口。

### 11.4 这节的大局：先验证靶，再建方法
可行性研判（`docs/FEASIBILITY_2026-06-23.md`）结论：别再和 PAM50 死磕单队列 OS（赢不了）；
换到**跨队列稳健**这个真缺口。但**先做几天的最小实验**量化"平台漂移有多大、现有方法跨队列差多少"——
差距大才值得投入。**先验证靶子值不值得打，再建大模型**——这正是我们前面跳过、导致绕弯十周的一步。

## 第 12 课 — 跨队列迁移 / 最近质心(PAM50 原理) / "饱和问题没有新意"（紧扣跨队列验证）

### 12.1 最近质心分类器 = PAM50 的本质
**PAM50 怎么工作**：给每个亚型算一个"质心"（50 基因的平均表达谱），新病人归到**最近的质心**。
我们的验证就用 `NearestCentroid`——这正是 PAM50 的原理，所以是最公平的"现有简单方法"基线。

### 12.2 跨队列迁移：z-score 为什么能消大部分平台差异
TCGA(RNA-seq) 与 METABRIC(芯片) 数值尺度不同。但**在每个队列内对每个基因 z-score**（减均值除标准差）后，
"相对高低"被对齐了——平台的一阶尺度差被消掉。结果：跨队列准确率(0.72-0.73) 几乎不输队列内(0.74)，**差距仅 0.02**。
**含义**：对 PAM50 这种"相对表达模式"的分型，平台漂移没想象中可怕，简单归一化就够。

### 12.3 最重要的元教训：饱和问题 = 没有创新空间
我们廉价地否掉了两个靶：① 超越 PAM50（W6-W10 都失败）② 跨队列稳健（这次，平凡方法已解决）。
模式很清楚——**TCGA-BRCA 分子分型是个"已解决/饱和"的问题**：PAM50 有效、预后好、能跨队列、连最朴素分类器都能复现。
**在领域最成熟、被研究最透的问题上找新意，注定低产。** 这不是我们方法不行，是**靶选在了红海**。
科研选题的硬道理：**要去"答案还不知道、现有方法还不行"的地方**，那里才有创新空间。下一步该重新选问题。

### 12.4 "先验证靶"这次省了几周
只花几小时 + 一个简单脚本，就否掉了"跨队列"这个本要建深度模型才试的方向。**先用最简单的基线探明靶值不值得打，
再投入**——这是这十几周最该内化的方法论。

## 第 13 课 — 三个候选新领域的入门生物学（紧扣换靶）

### 13.1 药物响应预测 (drug response prediction)
- **细胞系 (cell line)**：在实验室能无限培养的癌细胞群（如 MCF7）。可批量测它们对各种药的敏感性。
- **IC50**：抑制 50% 细胞所需的药物浓度——**越低=越敏感**。这是预测目标（回归）。
- **数据**：**GDSC**（~700 细胞系 × ~138 药的敏感性）、**DepMap/CCLE**（细胞系的表达/突变/拷贝数/基因依赖）。
- **任务（CS 视角）**：输入细胞系基因组特征 → 预测某药 IC50。纯监督回归，指标干净（Pearson/RMSE）。**最像传统 ML**。

### 13.2 空间转录组 (spatial transcriptomics, ST)
- **普通 RNA-seq 丢了位置**：把组织打碎测表达，不知道每个细胞在组织里**哪个位置**。
- **ST 保留位置**：在组织切片上按网格(spot)测表达，每个 spot 有 (x,y) 坐标 + 表达谱。代表平台 **10x Visium**。
- **核心任务**：**空间域识别**（把组织切成有生物学意义的区域，如肿瘤区/基质区/免疫浸润区）；**去卷积**（一个 spot 含多个细胞，估各细胞类型比例）；**多模态**（表达 + 配套的组织 H&E 图像）。
- **CS 视角**：spot 有位置 → 天然是**图(graph)**（相邻 spot 连边）→ **图神经网络(GNN)** 的主场。CS 优势直接发挥。较新(2021+)、较开放。

### 13.3 计算病理 × 基因组多模态
- **全切片图像 (WSI)**：把病理玻片扫成超高分辨率数字图（gigapixel，几十亿像素）。
- **任务**：从 WSI（+组学）预测生存/亚型/突变。**视觉 + 组学的多模态融合**。
- **CS 视角**：CV(图像) + 多模态，但 **gigapixel 数据极重**（显存/存储门槛高），且被大厂"基础模型"主导，个人难正面赢。故暂不推荐。

### 13.4 选靶的现实
**生信 ML 没有空白区，处处拥挤**——选 **CS 契合最高(ST)** 或 **任务最 ML、最易快速出结果(药物响应)** 的，
比去和领域专家拼生物学(病理/BRCA 分型)更适合一个 CS 背景的人。详见 `docs/CANDIDATE_TARGETS_2026-06-24.md`。

## 第 14 课 — 空间域识别 / DLPFC 基准 / 为什么"空间平滑"就有用（紧扣 ST 验证）

### 14.1 DLPFC 基准与任务
- **DLPFC**(人脑背外侧前额叶皮层)：大脑皮层有**分层结构**——L1、L2、…、L6 + 白质(WM)，共 7 层，**空间上连续成带**。
- **10x Visium** 在切片上按 ~55µm 的 spot 网格测表达；Maynard 2021 给每个 spot **人工标注了所属层**=金标准。
- **任务**：只用表达(+空间坐标) 把 spot 聚成这 7 层 → 与人工标注算 **ARI**。这是 ST 域识别的"MNIST"，几乎所有方法都在它上面比。

### 14.2 为什么"空间近邻平滑"白捡 +0.12 ARI
皮层分层是**空间连续**的：相邻 spot 极大概率属于同一层。所以**把每个 spot 的表达与其空间近邻做平均**，
能消掉单 spot 的测量噪声、强化"层"信号 → 非空间 0.20 → 简单空间 0.32。
**这就是 ST 的核心直觉**：表达 + **空间位置** 一起用，比只用表达强。CS 类比：给特征加了"空间先验/图结构正则"。

### 14.3 为什么这个靶"可做"而 BRCA 跨队列"不可做"
- BRCA 跨队列：朴素方法差距仅 **0.02** → 已饱和、没空间。
- ST 域识别：朴素方法距 SOTA **0.28** → 大空隙、有明确可量化的提升目标(0.32 → 0.60+)。
**"有没有开放空间"= 朴素基线离 SOTA 有多远**——这是选靶最实用的一把尺。

### 14.4 SOTA 怎么做的（下一步要复现并超越）
STAGATE/GraphST 等把 spot 建成**图**(相邻 spot 连边)，用**图注意力自编码器 / 图对比学习**联合学"既像邻居又保表达特征"的表示，再聚类 → ARI ~0.6。
这是图神经网络(GNN)的主场，正是 CS 的强项。下一步(ST-W2)：先复现一个这种 SOTA 级 GNN 基线(GPU)，再找创新点。

## 第 15 课 — GCN vs 图注意力(GAT) / 对比学习 / "用官方方法当基线"（紧扣 ST-W2）

### 15.1 图卷积(GCN)：让 spot 融合邻居
**GCN** 的一层 = `Â X W`：`Â` 是归一化的空间邻接矩阵，让每个 spot 的新表示 = 自身+邻居表示的加权平均(权重固定=按度归一化)。
我手搓的 **GCN 自编码器** 就是用它编码再重构表达。结果 ARI~0.33——比非空间(0.20) 好，但不及 SOTA(0.60)。

### 15.2 为什么朴素 GCN 不够，SOTA 强在哪
- **GAT(图注意力, STAGATE 用)**：邻居权重**不是固定的**，而是**学出来的**——模型自己决定"该多听哪个邻居"。
  在层边界处(两层交界的 spot)，能学会少听"另一层的邻居"，于是边界更干净 → ARI 更高。
- **对比学习(GraphST 用)**：让"相邻/相似 spot"表示靠近、"无关 spot"推远，学到更判别性的表示。
- 我实测**降学习率没用** → 说明瓶颈是**架构**(固定权重的 GCN)，不是调参。**朴素 GCN 天花板就在 ~0.33-0.45**。

### 15.3 科研规范：用"官方方法"当要打败的基线
手搓复现 SOTA 很难且易不达标(我就没搓到 0.6)。规范做法是**直接装官方实现(GraphST/STAGATE)**当对照——
(a) 确认在你这份数据上的真实天花板；(b) 你后续的"创新"必须和**官方真实结果**比，才有说服力。
手搓基线不达标是正常研究节奏，**用官方方法定基线不丢人**，反而更严谨。下一步就装 GraphST 复现 ~0.6。

### 15.4 一个工程教训(本轮踩的)
`conda run -n ENV python -c "多行脚本"` 会因参数含换行报错；多行脚本要**写成 .py 文件**再 `conda run python file.py`，
或用环境的 `python.exe` 全路径。`unzip "dir/*"` 的 `*` 不跨 `/`，子目录要显式列或用 Python `zipfile`。

## 第 16 课 — GraphST(SOTA) / 受控评估 / 研究三段式（紧扣 ST-W2 收尾）

### 16.1 GraphST 怎么拿到 ~0.6 的
GraphST = **图自监督对比学习**：在空间邻接图上，用**数据增强 + 对比损失**（让同一/相邻 spot 的两个增强视图表示靠近、无关 spot 推远）
学到判别性强的 spot 表示，再聚类。比我那个"只重构表达"的朴素 GCN-AE 强得多（0.54 vs 0.34）——对比学习 + 图，是当前 ST 的主流强招。

### 16.2 受控评估：所有方法用同一个聚类器
GraphST 论文报 ~0.63 用的是 **R 的 mclust**；我用 Python 的 GMM 得 ~0.54。差距来自**聚类器**，不是表示本身。
**科研要点**：比较不同方法时，要把"下游聚类器"固定成同一个，这样比的才是"表示(embedding)的好坏"，而不是"谁的聚类器调得好"。
所以我们的"要打败的对象"= **GraphST + 同一 GMM+精修 = 0.54**；我们将来的方法也用同一 GMM 比，才公平。

### 16.3 这十几周沉淀的"研究三段式"
1. **验证靶**：最简单基线离 SOTA 有多远？远=有空间(ST ✓)，近=饱和换靶(BRCA ✗)。
2. **立基线**：把 SOTA(官方实现)在自己数据上跑出真实天花板 = "要打败的对象"。
3. **找创新**：分析 SOTA 失效处 → 提出机制 → 在同一基准/受控评估下超越它。
现在 ST 走到了第 3 段的门口（前两段都扎实）。第 3 段才是真功夫，也是论文的命门。

### 16.4 空间域识别的评估基准(下一步要做全)
SOTA 论文报的是 **DLPFC 全 12 个样本的平均 ARI**(不是单样本)。所以要下任何"超越 SOTA"的结论，
必须在**12 样本**上跑(我们目前只在 151673)。这是 ST-W3 的第一步。
